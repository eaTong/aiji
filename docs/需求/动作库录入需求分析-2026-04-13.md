# 动作库录入需求分析

**创建时间**：2026-04-13
**需求方**：用户
**状态**：调研完成，待评审

---

## 一、需求背景

当前系统已有动作库查看功能（列表+详情），但缺少动作的创建、编辑、删除能力。用户需要能够：
1. 手动新增自定义动作
2. 编辑已有动作（复制后修改）
3. 批量导入预定义动作集

## 三、动作属性设计

基于现有 Exercise 模型（prisma/schema.prisma）：

### 3.1 完整属性列表

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | ✅ | 动作中文名称 |
| nameEn | string | ❌ | 动作英文名称 |
| category | enum | ✅ | 动作分类：CHEST/BACK/LEGS/SHOULDERS/ARMS/CORE/CARDIO |
| equipment | enum | ✅ | 器械类型：GYM/DUMBBELL/BODYWEIGHT |
| primaryMuscles | Json | ✅ | 主肌群数组，如 ["胸大肌", "三角肌前束"] |

| isCustom | boolean | 系统生成 | 是否自定义动作 |
| isFavorite | boolean | 系统生成 | 是否收藏 |

## 四、录入方式设计

### 4.1 手动录入

**触发入口**：动作库列表页右上角「+」按钮

**流程**：
```
动作库列表页 → [+] → 新增动作表单 → 填写信息 → 保存 → 返回列表
```

### 4.2 编辑已有动作

**触发入口**：动作详情页右上角「编辑」按钮

**流程**：
```
动作详情页 → [编辑] → 编辑表单（预填充数据） → 保存 → 返回详情
```

**注意**：系统动作（isCustom=false）不允许直接编辑，需复制后创建自定义版本

### 4.3 批量导入

**触发入口**：动作库列表页「批量导入」按钮

**流程**：
```
动作库列表页 → [批量导入] → 选择导入模板/导入数据 → 确认 → 执行导入
```

## 六、接口设计

### 6.1 创建动作
```
POST /api/exercise
Body: {
  name: string,
  nameEn?: string,
  category: ExerciseCategory,
  equipment: Equipment,
  primaryMuscles: string[],
  secondaryMuscles?: string[],
  instructions?: string,
  commonMistakes?: string,
  videoUrl?: string
}
Response: Exercise
```

### 6.2 更新动作
```
PUT /api/exercise/:id
Body: Partial<Exercise>
Response: Exercise
```

### 6.3 删除动作
```
DELETE /api/exercise/:id
Response: { success: true }
限制：仅 isCustom=true 的动作可删除
```

### 6.4 批量导入
```
POST /api/exercise/batch
Body: {
  exercises: Exercise[]
}
Response: { imported: number, skipped: number }
```

### 8.3 备选方案：wger 项目

**项目地址**：https://github.com/wger-project/wger

**数据规模**：1000+ 个动作

**License**：GNU Affero General Public License (AGPL-3.0) - 开源有传染性

**字段覆盖度**：

| 用户需求字段 | wger 字段 | 覆盖情况 |
|-------------|-----------|---------|
| 动作名称 | `Translation.name` | ✅ |
| 动作说明 | `Translation.description` | ✅ (min 40 chars) |
| 注意事项 | `ExerciseComment.comment` | ✅ 有专门评论字段 |
| 易错点 | - | ❌ 缺失 |
| 主肌群 | `Exercise.muscles` | ✅ |
| 辅肌群 | `Exercise.muscles_secondary` | ✅ |
| 难度 | - | ❌ 缺失 |

**数据结构**：
```
Exercise (base)
├── uuid
├── category → ExerciseCategory
├── muscles → ManyToMany(Muscle) [主肌群]
├── muscles_secondary → ManyToMany(Muscle) [辅肌群]
├── equipment → ManyToMany(Equipment)
└── translations → Translation[]
    ├── name (多语言)
    ├── description (min 40 chars)
    └── exercisecomment_set → ExerciseComment[] (注意事项)
```

**肌肉分类**（支持前后侧区分）：
```json
{
  "name": "Biceps brachii",
  "is_front": true,
  "name_en": "Biceps"
}
```

**优劣势分析**：

| 优势 | 劣势 |
|------|------|
| ✅ 数据最完整，有"注意事项"字段 | ❌ AGPL License 有传染性 |
| ✅ 多语言支持（英语、德语等） | ❌ 数据需通过 Django ORM 获取 |
| ✅ 社区活跃，1000+ 动作 | ❌ 缺少难度等级字段 |
| ✅ 支持肌肉前后侧区分 | ❌ 不适合直接导入商业项目 |

**AGPL 传染性说明**：
- 如果修改 wger 代码或衍生作品，需开源
- 但导入其数据（ fixtures ）作为参考应该是可以的
- 建议：仅用 wger 数据作为肌肉名称映射参考

### 8.5 3D 动画数据源调研结论

经过全面调研，**目前不存在免费且可商用的 3D 训练动画库**。调研结果如下：

| 数据源 | 类型 | License | 免费 | 商用 | 推荐 |
|--------|------|---------|------|------|------|
| free-exercise-db | 动作数据 | Public Domain | ✅ | ✅ | ✅ **推荐** |
| Mixamo (Adobe) | 3D 动画 | Adobe Terms | ✅ | ❌ 需 Creative Cloud | ❌ |
| exerciseanimatic.com | 3D 动画视频 | 商业授权 | ❌ | ✅（付费） | ❌ |
| rendu.vp | 3D 动画 | 商业授权 | ❌ | ✅（付费） | ❌ |
| RenderForest | 3D 动画 | 商业授权 | ❌ | ✅（付费） | ❌ |

**免费方案分析**：

1. **free-exercise-db + 自建图片/视频**
   - 数据免费（800+动作）
   - 肌肉图解可自制或购买版权图片
   - 视频需自录制或购买

2. **Mixamo 免费层**
   - 需要 Adobe Creative Cloud 订阅
   - 3D 模型可导出，但动画版权受限

3. **付费方案**（可选）
   - exerciseanimatic.com：$1/动作，$599 终身 bundle
   - 适合预算充足、追求高品质的用户

**最终推荐**：
- **数据源**：free-exercise-db（免费，Public Domain）
- **视觉内容**：初期使用肌肉图解代替视频，中期可考虑付费采购视频

### 8.7 导入策略

1. **数据源选择**：使用 free-exercise-db（800+动作，Public Domain）
2. **肌肉名称映射**：建立中英文映射表，批量转换
3. **分类映射**：
   - `strength` → CHEST/BACK/LEGS/ARMS 等
   - `cardio` → CARDIO
   - `stretching` → CORE
4. **用户自定义优先**：用户自定义动作显示在系统动作之前
