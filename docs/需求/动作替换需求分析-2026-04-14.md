# 动作替换需求分析

> **日期：** 2026-04-14
> **功能：** Phase 2B 补充 - 训练计划动作替换
> **优先级：** P0

---

## 一、背景与目标

### 1.1 问题

用户在使用 AI 生成的训练计划时，可能会对某些动作不满意：
- 不感兴趣的动作
- 没有相应器材
- 不知道如何正确练习

用户需要一个便捷的方式来替换计划中的动作，同时系统需要记录替换原因以便后续优化。

### 1.2 目标

- 用户可在计划详情页点击动作卡片发起替换
- 提供明确的替换原因选项（不感兴趣、没有器械、不会练、其他）
- 显示同类型可替换动作供用户选择
- 记录替换历史用于数据分析和优化

## 三、数据模型

### 3.1 PlanExerciseReplacement 模型

**文件：** `backend/prisma/schema.prisma`

```prisma
model PlanExerciseReplacement {
  id                  String   @id @default(uuid())
  plannedExerciseId   String   // 被替换的计划动作 ID
  originalExerciseId  String   // 原动作 ID
  newExerciseId       String   // 新动作 ID
  reason             String   // 替换原因
  createdAt          DateTime @default(now())

  // 关系
  plannedExercise    PlannedExercise @relation(fields: [plannedExerciseId], references: [id])
}

model PlannedExercise {
  // ... existing fields ...
  replacements PlanExerciseReplacement[]  // 反向关系
}
```

### 3.2 替换历史用途

- 分析用户替换原因分布，优化动作推荐算法
- 识别不受欢迎的动作，持续优化动作库
- 支持用户查看自己的替换历史

## 四、API 设计

### 4.1 获取可替换动作列表

**端点：** `GET /api/training-plans/:planId/exercises/:exerciseId/replacable`

**请求参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| planId | string | 计划 ID |
| exerciseId | string | 计划动作 ID（plannedExerciseId） |

**响应：**
```typescript
interface ReplaceableExercisesResult {
  currentExercise: Exercise           // 当前动作详情
  replacableExercises: Exercise[]    // 可替换动作列表
}
```

**可替换动作筛选逻辑：**
1. 同 category（胸部/背部/腿部/肩部/手臂/核心/有氧）
2. 满足用户器材条件（equipment 兼容）
3. 排除原动作本身

### 4.2 执行替换

**端点：** `PUT /api/training-plans/:planId/exercises/:exerciseId/replace`

**请求参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| planId | string | 计划 ID |
| exerciseId | string | 计划动作 ID |
| newExerciseId | string | 新动作 ID |
| reason | string | 替换原因 |

**请求体：**
```typescript
interface ReplaceExerciseParams {
  newExerciseId: string
  reason: 'not_interested' | 'no_equipment' | 'dont_know_how' | 'other'
}
```

**响应：**
```typescript
interface ReplaceExerciseResponse {
  success: boolean
  replacement: PlanExerciseReplacement  // 替换记录
  updatedExercise: PlannedExercise       // 更新后的计划动作
}
```

  await loadPlan()  // 重新加载计划数据
}

### 7.2 Controller 函数

**文件：** `backend/src/controllers/trainingPlanController.ts`

```typescript
export async function getReplacableExercises(ctx: Context) {
  const { exerciseId } = ctx.params
  const result = await getReplacableExercises(exerciseId)
  ctx.body = success(result)
}

export async function replaceExercise(ctx: Context) {
  const { exerciseId } = ctx.params
  const { newExerciseId, reason } = ctx.request.body as {
    newExerciseId: string
    reason: string
  }

  if (!newExerciseId || !reason) {
    ctx.status = 400
    ctx.body = { code: 400, message: '缺少必填字段', data: null }
    return
  }

  const result = await replaceExercise({
    plannedExerciseId: exerciseId,
    newExerciseId,
    reason: reason as ReplacementReason,
  })
  ctx.body = success(result)
}
```

## 九、验收标准

### 9.1 功能验收

- [ ] 点击动作卡片可打开替换弹窗
- [ ] 替换原因 4 个选项可正常选择
- [ ] 可替换动作列表正确显示同类型动作
- [ ] 选择原因和动作后可点击确认
- [ ] 替换成功后界面刷新显示新动作
- [ ] 替换历史正确记录到数据库

### 9.2 测试场景

| 场景 | 预期结果 |
|------|----------|
| 点击动作卡片 | 弹窗打开，显示当前动作信息 |
| 选择"不感兴趣" | 原因被选中，可替换列表正常加载 |
| 选择新动作 + 确认 | 替换成功，界面刷新，数据显示新动作 |
| 替换历史 | 数据库 PlanExerciseReplacement 表有新记录 |

### 9.3 异常处理

| 场景 | 预期结果 |
|------|----------|
| 可替换动作为空 | 显示"没有可替换的动作"提示 |
| 网络请求失败 | 显示错误 Toast，弹窗不关闭 |
| 未选择原因或动作 | 确认按钮禁用，不可点击 |
