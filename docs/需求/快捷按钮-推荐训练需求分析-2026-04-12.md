# 快捷按钮「不知道练什么？推荐一下」需求分析

**日期**：2026-04-12
**版本**：v1.0
**状态**：已确认
**类型**：需求拓展分析

---

## 一、背景与目的

### 1.1 背景回顾

根据现有设计文档（docs/superpowers/specs/2026-04-12-aiji-miniprogram-design.md）：

**AI对话框定位**：
- AI对话是唯一主入口，所有功能通过对话触达
- 快捷操作栏位于对话框上方，现有5个快捷按钮：记体重、记训练、记饮食、记围度、问教练

**训练模块（Tab 3）现有能力**：
- 我的计划：AI生成训练计划（基于用户档案：健身目标/当前水平/可用器材/每周可训练天数）
- 今日训练：动作列表执行界面
- 动作库：分类浏览/动作搜索
- 恢复状态：恢复评分/肌群恢复热力图/训练建议

### 1.2 新增快捷按钮目的

**用户场景**：用户打开AI对话框，想训练但不知道练什么，希望快速获得一个训练推荐

**设计目标**：降低用户决策门槛，一键触达个性化训练推荐

## 二、核心问题分析

### Q1: 点击后的触发流程（AI如何响应）

**触发流程设计**：

```
用户点击「不知道练什么？推荐一下」
         ↓
系统收集上下文数据（异步，不阻塞UI）
         ↓
AI 生成训练推荐（对话流形式）
         ↓
展示推荐结果（卡片+对话）
         ↓
用户确认/调整/执行
```

**AI 响应内容结构**：

```
AI：「根据你的情况，今天推荐练【部位/类型】」
      ↓
【推荐训练卡片】
- 训练名称
- 预计时长
- 训练内容摘要（主要动作3-5个）
- 适配原因（简短1-2句话说明为何推荐这个）
      ↓
「要开始练吗？」或「换一个推荐」
```

### Q2: 推荐逻辑依据（应考虑哪些因素）

**核心因素优先级排序**：

| 优先级 | 因素 | 数据来源 | 说明 |
|-------|------|---------|------|
| P0 | 用户健身目标 | 个人档案（减脂/增肌/塑形/提升体能） | 决定训练类型（力量 vs 有氧 vs 混合） |
| P0 | 可用器材 | 个人档案（健身房/哑铃/徒手） | 决定可推荐的动作范围 |
| P1 | 训练历史（近7天） | 训练记录数据库 | 避免连续重复同一肌群 |
| P1 | 当前恢复状态 | 恢复评分/肌群恢复热力图 | 避免过度训练未恢复的肌群 |
| P1 | 今日计划状态 | 训练计划表 | 若今日已有计划，直接展示现有计划 |
| P2 | 距离上次训练天数 | 训练记录数据库 | 判断是否需要主动推动训练 |
| P2 | 用户偏好（动作历史） | 训练记录数据库 | 倾向于用户常做的动作类型 |
| P3 | 当日时间 | 系统时间 | 晚间推荐轻度训练，早晨推荐常规训练 |

**推荐算法伪逻辑**：

```
function generateRecommendation(userContext):
    // 1. 检查今日是否有已计划训练
    todayPlan = getTodayPlan(userContext.userId)
    if todayPlan and todayPlan.status == 'pending':
        return { type: 'existing_plan', plan: todayPlan }

    // 2. 获取未充分恢复的肌群（避免推荐）
    recoveredMuscles = getRecoveredMuscles(userContext.userId)

    // 3. 根据健身目标确定训练类型
    if userContext.goal == '减脂':
        trainingType = randomChoice(['力量+有氧', 'HIIT'])
    else if userContext.goal == '增肌':
        trainingType = '力量训练'
    else:
        trainingType = randomChoice(['力量', '混合'])

    // 4. 获取近7天训练部位，避免重复
    recentTrainedMuscles = getRecentTrainedMuscles(userContext.userId, days=7)

    // 5. 结合器材约束，筛选可用动作
    availableExercises = filterByEquipment(userContext.equipment, allExercises)

    // 6. 选择目标肌群（优先选近期未训练+已恢复的）
    targetMuscle = selectMuscle(recentTrainedMuscles, recoveredMuscles, trainingType)

    // 7. 生成训练内容
    exercises = selectExercisesForMuscle(targetMuscle, availableExercises, count=4-6)

    return {
        type: 'new_recommendation',
        trainingType: trainingType,
        targetMuscle: targetMuscle,
        exercises: exercises,
        reason: generateReason(targetMuscle, userContext)
    }
```

### Q4: 与现有「训练计划生成」功能的关系

**关系判定：独立但协同**

| 维度 | 训练计划生成（Tab 3） | 快捷按钮推荐 |
|-----|----------------------|-------------|
| 触发方式 | 用户主动进入Tab 3创建 | 快捷按钮一键触发 |
| 粒度 | 周期性计划（周计划） | 单次训练推荐 |
| 周期 | 4-8周完整周期 | 即时推荐 |
| 上下文 | 需要完整用户档案 | 可基于轻量上下文快速响应 |
| 用户确认 | 创建计划需要确认 | 推荐可直接执行或跳过 |

**协同策略**：

1. **入口独立**：快捷按钮提供快速临时推荐，不影响正式计划
2. **数据共享**：复用自己的计划动作库、动作数据
3. **计划融合**：若用户通过快捷按钮完成训练，自动记录并更新计划进度
4. **计划优先**：当用户有进行中的计划时，优先推荐计划内容而非随机推荐

**复用设计点**：
- 动作库（同一套动作数据）
- 用户档案信息（目标/器材/水平）
- 训练记录存储（同一数据库）
- 恢复状态计算（同一算法）

### Q5: 推荐的训练内容深度（是否需要用户确认）

**设计决策：轻确认，可直接执行**

**推荐内容深度**：
- 动作数量：4-6个动作（精简版，约45分钟）
- 重量/次数：不预设，使用「上次重量参考」或「AI建议起始重量」
- 组间休息：不预设，使用标准休息时间

**确认流程（可选）**：

```
快捷推荐 → 用户点击「开始训练」
         ↓
进入「今日训练」执行界面
         ↓
动作列表预填，但需要用户：
- 确认重量（可调整）
- 每组完成后记录实际次数
         ↓
训练完成后 AI 总结
```

**简化路径**：用户可直接跳过确认，使用「快速训练」模式（不记录重量/次数，仅记录完成）

### Q6: 异常场景处理

#### 场景1：用户已完成今日训练

**触发条件**：今日已有训练记录（status=completed）

**AI 响应**：
```
「你今天已经练过了！训练记录：胸部力量训练 45分钟

要不要休息一天让肌肉恢复？或者我可以给你安排一些拉伸/轻度有氧。」
```

**按钮**：[安排拉伸] [轻度有氧] [查看周计划]

---

#### 场景2：用户没有设置健身目标

**触发条件**：用户档案缺少健身目标（goal=null）

**AI 响应**：
```
「要给你推荐训练，先了解一下你的健身目标~

你现在的主要目标是？」
```

**按钮**：[减脂] [增肌] [塑形] [提升体能]

**后续**：选择后记录到用户档案，再继续推荐流程

---

#### 场景3：用户没有可用训练器材信息

**触发条件**：用户档案器材设置为空

**AI 响应**：
```
「你有哪些训练器材可用？」
```

**按钮**：[健身房] [哑铃+杠铃] [徒手]

**处理**：选择后记录到用户档案，再继续推荐流程

---

#### 场景4：所有肌群都未恢复（过度训练预警）

**触发条件**：肌群恢复热力图显示所有主要肌群恢复状态 < 50%

**AI 响应**：
```
「你的身体需要休息！所有肌群恢复状态都不太理想。

建议今天安排休息或轻度活动：
• 散步30分钟
• 拉伸15分钟
• 泡沫轴放松」
```

**按钮**：[休息建议] [拉伸动作] [查看恢复状态]

---

#### 场景5：训练计划已满（今日已安排训练）

**触发条件**：今日训练计划已存在且状态=pending

**AI 响应**：
```
「你今天有计划训练：背部力量训练

要按计划执行吗？还是换一个？」
```

**按钮**：[执行计划] [换一个] [调整计划]

---

#### 场景6：新用户（无任何训练记录）

**触发条件**：训练记录为空

**AI 响应**：
```
「欢迎开始健身！根据你的目标【增肌】，推荐从基础动作开始：

今日训练：全身力量（新手友好）
• 深蹲 3×12
• 俯身划船 3×12
• 卧推 3×12
• 肩推 3×12

这些动作覆盖全身主要肌群，适合刚开始健身的你。」
```

**按钮**：[开始训练] [了解动作要点]

---

#### 场景7：网络异常/系统错误

**AI 响应**：
```
「抱歉，获取推荐时遇到了一点问题，请稍后再试~

或者直接去【训练】Tab 查看你的计划？」
```

**按钮**：[重试] [查看训练Tab]

## 四、AI 响应逻辑

### 4.1 推荐生成 Prompt 核心片段

```
你是健身助手"AI己"，根据用户情况生成一次训练推荐。

用户上下文：
- 健身目标：{goal}
- 当前水平：{level}
- 可用器材：{equipment}
- 上次训练：{lastTraining}
- 肌群恢复状态：{recoveryStatus}
- 今日状态：{todayStatus}

输出要求：
1. 推荐一个训练部位/类型（简洁）
2. 列出4-6个动作（考虑器材约束）
3. 说明推荐原因（1-2句话，结合用户实际情况）
4. 预估时长
5. 语气亲切自然，像朋友建议而非系统输出
```

### 4.2 边界场景 Prompt 策略

| 场景 | Prompt 策略 |
|-----|-----------|
| 已完成今日训练 | 「夸赞+休息建议」而非再次推荐 |
| 无目标/器材 | 「引导式提问」而非拒绝 |
| 过度训练预警 | 「关心+强制休息建议」 |
| 新用户 | 「鼓励+简化版全身训练」 |

## 七、待确认事项

1. **推荐次数限制**：同一会话内"换一个"最多支持几次？（建议3次）
2. **历史推荐记录**：是否需要记录用户拒绝的推荐，避免重复推荐？
3. **快捷推荐与计划的冲突**：当用户有进行中计划时，优先展示计划还是独立推荐？
4. **数据完整性要求**：无目标/器材时，是引导设置（阻塞式）还是使用默认值继续？

## 八、优先级建议

### Phase 1（MVP）- 推荐训练快捷按钮

**核心功能**：
1. 快捷按钮入口（位于现有快捷操作栏）
2. 基础推荐逻辑（基于目标+器材+训练历史）
3. 正常场景处理（已完成训练/无目标/新用户）
4. 推荐卡片展示
5. 开始训练/换一个/查看计划 三个交互按钮

**不纳入MVP**：
- 过度训练预警（需要肌群恢复数据完整）
- 智能恢复状态判断

### Phase 2（增强）

1. 肌群恢复热力图接入
2. 过度训练预警场景
3. 快速训练模式（不记录重量）
