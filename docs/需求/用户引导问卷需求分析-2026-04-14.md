# 用户引导问卷需求分析

> **日期：** 2026-04-14
> **功能：** Phase 2B 补充 - 用户引导问卷 + 智能计划生成
> **优先级：** P0

---

## 一、背景与目标

### 1.1 问题

当前训练计划生成依赖用户档案（goal, level, equipment, weeklyTrainingDays），但新用户没有填写过档案。需要在生成计划前引导用户完成基本信息收集，然后使用现有规则引擎生成个性化训练计划。

### 1.2 目标

- 新用户首次打开小程序时，引导完成基本信息收集
- 基于用户信息自动生成个性化训练计划
- 已完成引导的用户可跳过问卷直接生成计划

## 三、问卷流程设计

### 3.1 Step 1：健身目标

**问题：** 你健身的主要目标是什么？

| 选项 | 值 | 说明 |
|------|-----|------|
| 增肌 | GAIN_MUSCLE | 增加肌肉量 |
| 减脂 | LOSE_FAT | 减少体脂 |
| 塑形 | BODY_SHAPE | 改善体型 |
| 提升体能 | IMPROVE_FITNESS | 提高整体体能 |

### 3.2 Step 2：当前状态

**问题 A：** 你的训练水平是？

| 选项 | 值 | 说明 |
|------|-----|------|
| 初学者 | BEGINNER | 健身经验不足 6 个月 |
| 中级 | INTERMEDIATE | 6-12 个月训练经验 |
| 高级 | ADVANCED | 超过 1 年训练经验 |

**问题 B：** 你有哪些器材可用？

| 选项 | 值 | 说明 |
|------|-----|------|
| 健身房 | GYM | 完整器械 |
| 哑铃 | DUMBBELL | 哑铃 + 杠铃 |
| 自重 | BODYWEIGHT | 无额外器材 |

**问题 C：** 你每周能训练几天？

| 选项 | 值 | 说明 |
|------|-----|------|
| 1-7 天 | weeklyTrainingDays | 用户选择具体数字 |

### 3.3 Step 3：身体数据（可选跳过）

**问题：** 你的身体数据（选填，可跳过）

| 字段 | 类型 | 说明 |
|------|------|------|
| 身高 | number (cm) | 选填 |
| 当前体重 | number (kg) | 选填，如提供则记录到 BodyData |
| 目标体重 | number (kg) | 选填 |

**跳过逻辑：** 用户可选择"跳过"，不传这些字段

### 3.4 Step 4：健身时长

**问题：** 你健身多久了？

| 选项 | 值 | 说明 |
|------|-----|------|
| 3个月以下 | LESS_THAN_3M | 刚开始 |
| 3-6个月 | THREE_TO_6M | 有一定基础 |
| 6-12个月 | SIX_TO_12M | 比较熟练 |
| 1年以上 | OVER_1Y | 经验丰富 |

## 五、API 设计

### 5.1 提交问卷答案

**端点：** `POST /api/onboarding/complete`

**请求体：**
```typescript
interface OnboardingCompleteBody {
  goal: 'GAIN_MUSCLE' | 'LOSE_FAT' | 'BODY_SHAPE' | 'IMPROVE_FITNESS'
  level: 'BEGINNER' | 'INTERMEDIATE' | 'ADVANCED'
  equipment: 'GYM' | 'DUMBBELL' | 'BODYWEIGHT'
  weeklyTrainingDays: number  // 1-7
  height?: number  // cm，选填
  currentWeight?: number  // kg，选填
  targetWeight?: number  // kg，选填
  fitnessDuration: 'LESS_THAN_3M' | 'THREE_TO_6M' | 'SIX_TO_12M' | 'OVER_1Y'
}
```

**响应：**
```typescript
interface OnboardingCompleteResponse {
  profileUpdated: boolean   // 档案是否更新成功
  weightRecorded: boolean   // 体重是否记录
  plan?: WorkoutPlan        // 生成的计划
}
```

### 5.2 获取引导状态

**端点：** `GET /api/onboarding/status`

**响应：**
```typescript
interface OnboardingStatusResponse {
  hasCompletedOnboarding: boolean
}
```

## 七、业务逻辑

### 7.1 问卷提交后处理

```typescript
// onboardingService.ts
async function processOnboarding(userId: string, data: OnboardingData) {
  // 1. 更新用户档案
  await userService.updateOnboardingProfile(userId, {
    goal: data.goal,
    level: data.level,
    equipment: data.equipment,
    weeklyTrainingDays: data.weeklyTrainingDays,
    height: data.height,
    targetWeight: data.targetWeight,
    fitnessDuration: data.fitnessDuration,
  })

  // 2. 记录体重（如果提供了）
  if (data.currentWeight) {
    await bodyDataService.recordWeight(userId, data.currentWeight)
  }

  // 3. 标记引导完成
  await userService.markOnboardingComplete(userId)

  // 4. 生成训练计划
  const plan = await trainingPlanService.generateTrainingPlan(userId)

  return plan
}
```

### 7.2 首次启动检测

```typescript
// front/pages/index/index.vue (首页)
onMounted(async () => {
  await userStore.fetchProfile()

  }
```

## 九、验收标准

### 9.1 功能验收

- [ ] 新用户首次打开小程序弹出引导蒙层
- [ ] 完成问卷 4 步后档案更新成功
- [ ] 提供了当前体重时记录到 BodyData 表
- [ ] 问卷提交后自动生成训练计划
- [ ] 生成成功后跳转计划详情页
- [ ] 已引导用户不再弹蒙层
- [ ] 已引导用户点击"生成计划"直接生成

### 9.2 测试场景

| 场景 | 预期结果 |
|------|----------|
| 新用户完整流程 | 未完成引导 → 弹蒙层 → 问卷 4 步 → 档案更新 → 计划生成 → 跳转详情 |
| 跳过身体数据 | Step 3 选择跳过 → 不传 currentWeight → 计划仍正常生成 |
| 重复引导 | 已完成引导的用户再次进入引导页 → 提示"已完成初始化" → 跳转计划列表 |
