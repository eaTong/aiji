## 二、功能流程

### 2.1 主流程

```
用户点击「不知道练什么？推荐一下」
         ↓
系统收集上下文（目标/器材/训练历史/恢复状态）
         ↓
AI 生成训练推荐 → 展示推荐卡片
         ↓
用户决策：[开始训练] / [换一个] / [查看计划]
```

### 2.2 数据输入

| 数据项 | 来源 | 必填 |
|-------|------|------|
| 健身目标 | 用户档案 | 是（缺失时引导设置） |
| 可用器材 | 用户档案 | 是（缺失时引导设置） |
| 训练历史（近7天） | 训练记录数据库 | 否 |
| 肌群恢复状态 | 恢复状态模块 | 否 |
| 今日计划状态 | 训练计划表 | 否 |

### 2.3 异常场景处理

| 场景 | AI 响应 | 按钮 |
|-----|---------|------|
| 今日已完成训练 | 夸赞+休息建议 | [安排拉伸] [轻度有氧] |
| 无健身目标 | 引导式提问 | [减脂] [增肌] [塑形] [提升体能] |
| 无器材信息 | 引导式提问 | [健身房] [哑铃+杠铃] [徒手] |
| 新用户无记录 | 全身训练推荐 | [开始训练] |
| 网络异常 | 友好提示 | [重试] [查看训练Tab] |

## 四、后台接口定义

### 4.1 获取训练推荐

**接口**：`POST /api/training/recommend`

**请求参数**：
```json
{
  "userId": "string",
  "context": {
    "forceRefresh": false
  }
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "type": "new_recommendation" | "existing_plan" | "rest_day",
    "training": {
      "name": "胸部力量训练",
      "duration": 45,
      "type": "增肌",
      "targetMuscle": "chest",
      "exercises": [
        {
          "name": "杠铃卧推",
          "sets": 4,
          "reps": "8-10",
          "lastWeight": 80
        }
      ],
      "reason": "你已经3天没练上肢了，胸大肌恢复情况良好"
    }
  }
}
```

### 4.2 场景响应

**无目标时响应**：
```json
{
  "code": 0,
  "data": {
    "type": "need_goal",
    "message": "要给你推荐训练，先了解一下你的健身目标~",
    "options": ["减脂", "增肌", "塑形", "提升体能"]
  }
}
```

**已完成今日训练响应**：
```json
{
  "code": 0,
  "data": {
    "type": "completed_today",
    "message": "你今天已经练过了！",
    "completedTraining": {
      "name": "胸部力量训练",
      "duration": 45
    },
    "suggestions": ["安排拉伸", "轻度有氧"],
    "options": ["安排拉伸", "轻度有氧", "查看周报"]
  }
}
```
