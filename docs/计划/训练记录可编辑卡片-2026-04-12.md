### 2.2 字段编辑方式

| 字段 | 交互方式 | 控件类型 |
|-----|---------|---------|
| 动作名称 | 点击下拉 | 下拉选择器 + 搜索 |
| 组数 | 点击编辑 | 数字输入框（步进器） |
| 次数 | 点击编辑 | 数字输入框 |
| 重量 | 点击编辑 | 数字输入框 + 单位切换 |
| 热身组 | 点击切换 | 圆形按钮 ●/○ |
| 操作 | 点击 | 删除按钮（红色） |

### 2.3 热身行样式

```
• 行背景：rgba(255,152,0,0.08)（浅橙色）
• 动作名称后缀：" (热身)"
• 热身组列：●（实心圆）
```

## 三、数据模型

### 3.1 前端状态

```typescript
interface ExerciseSet {
  id: string;                    // 唯一标识（UUID）
  exerciseId: string;            // 动作ID
  exerciseName: string;          // 动作名称
  sets: number;                  // 组数
  reps: number;                  // 次数
  weight: number;               // 重量
  weightUnit: 'kg' | 'lb';      // 重量单位
  isWarmup: boolean;             // 是否热身组
}

interface TrainingRecordDraft {
  date: string;                  // YYYY-MM-DD
  trainingType: string;          // 训练类型
  exercises: ExerciseSet[];      // 动作列表
  notes: string;                 // 备注
}
```

### 3.2 后端接口

**保存训练记录**：`POST /api/training/record`

**请求**：
```json
{
  "date": "2026-04-12",
  "trainingType": "胸部训练",
  "exercises": [
    {
      "exerciseId": "uuid",
      "exerciseName": "杠铃卧推",
      "sets": 4,
      "reps": 10,
      "weight": 60,
      "weightUnit": "kg",
      "isWarmup": false
    }
  ],
  "notes": "状态不错"
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "id": "record-uuid",
    "totalVolume": 2400,
    "e1rm": {
      "杠铃卧推": 75
    },
    "message": "已记录！胸部力量训练，容量2400kg"
  }
}
```

## 四、交互流程

### 4.1 主流程

```
用户输入训练描述 → AI解析 → 展示可编辑卡片 → 用户编辑 → 保存 → AI反馈
```

### 4.2 卡片状态机

```
草稿态 → 用户编辑 → 确认保存 → 保存中 → 成功/失败
                         ↓
                      [取消] → 卡片销毁
```

## 五、热身组自动判断

### 5.1 判断规则

| 规则 | 条件 | 结果 |
|-----|------|------|
| 规则1 | 第一组 + 重量 < 历史最大重量50% | 热身组 |
| 规则2 | 第一组 + 次数 > 15 | 热身组 |
| 规则3 | 重量 < 历史最大重量30% | 热身组 |
| 用户修正 | 用户手动切换 | 优先于规则 |

### 5.2 视觉区分

- 热身组行：浅橙色背景 (#FFF8E1)
- 动作名称后显示 "(热身)"
- 热身组不计入总训练容量

## 六、异常场景处理

| 场景 | AI 响应 |
|-----|---------|
| 完全解析失败 | 引导式提问 + 手动录入入口 |
| 部分解析成功 | 卡片展示 + 缺省字段高亮提示 |
| 动作识别模糊 | 候选选择卡片让用户确认 |
| 数据不合逻辑 | 提示修正，保留编辑能力 |
