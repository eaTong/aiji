## 工作流程

### 1. 需求阶段

**需求拓展**：当用户提出需求时，先进行需求拓展分析，调用 brainstorm 能力对需求进行规划，输出需求分析结果并同步到 `docs/需求/` 目录。

**需求定义**：在需求拓展完成后，定义具体内容：
- 具体页面内容
- 显示字段
- 操作流程
- 同步到 `docs/需求/` 目录

### 2. 计划阶段

读取需求文档和现有计划文档，进行规划：
- 完成具体页面结构
- 定义操作流程
- 输出操作流程文档
- 后台定义接口（无需实现代码）
- 计划文档保存到 `docs/计划/` 目录

### 3. 开发阶段（严格按顺序执行）

1. 开发后台逻辑
2. 完成后台单元测试
3. 开发前端界面并根据后台接口实现功能
4. 定义小程序自动化测试
5. 验证操作逻辑是否与需求规划阶段的操作流程一致

### 4. 需求修改流程

**必须严格遵循**：修改需求文档 → 确认需求文档 → 生成计划 → 根据计划执行
